# Memory Hierarchy Simulation Project

## Overview
This project demonstrates the concept of **memory hierarchy in computer architecture**, including how different levels of memory affect performance and data access time.  

The program simulates memory access and helps analyze the behavior of different memory levels such as **cache, main memory, and secondary storage**.

This project was developed as part of a **university coursework assignment** to understand how memory hierarchy improves system performance by reducing average memory access time.

---

## Objectives
The main objectives of this project are:

- Understand the concept of **memory hierarchy**
- Simulate memory access patterns
- Analyze **cache hit and miss behavior**
- Observe how memory levels affect system performance
- Study the importance of cache memory in modern processors

---

## Memory Hierarchy Concept

Modern computer systems use multiple levels of memory to balance **speed, cost, and storage capacity**.

Typical hierarchy:

1. CPU Registers
2. Cache Memory (L1, L2, L3)
3. Main Memory (RAM)
4. Secondary Storage (SSD / HDD)

Faster memory is smaller and more expensive, while slower memory provides larger storage.
